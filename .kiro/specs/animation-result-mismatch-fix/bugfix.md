# バグ修正要件ドキュメント

## はじめに

Discordルーレットアプリケーションにおいて、アニメーション表示と実際の選出結果が一致しない2つの重大なバグが報告されています。これらのバグは、ユーザーがアニメーション結果を信頼できなくなるため、アプリケーションの信頼性に深刻な影響を与えています。

**バグ1: 円形ルーレットアニメーションの結果不一致**
複数人を選出する際、各回のアニメーションで針が指したユーザーと、最終的に表示される選出結果が異なる問題が発生しています。例えば、2人選出時にアニメーションでは「太郎」と「花子」が選ばれたように見えるが、結果画面では「太郎」と「次郎」が表示されるといった不一致が発生します。

**バグ2: スロットマシンアニメーションの左端スロット停止問題**
スロットマシンアニメーション実行時、一番左のスロットが完全に静止したまま回転せず、アニメーション開始時から結果が確定してしまう問題が発生しています。これにより、スロットマシンアニメーションの演出効果が損なわれています。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 円形ルーレットアニメーションで1人以上を選出する場合 THEN 各回のアニメーションで針が指すユーザーと最終的な選出結果が一致しない

1.2 WHEN 円形ルーレットアニメーションで1回目のアニメーションが完了した後、2回目以降のアニメーションを実行する場合 THEN 前回のアニメーション結果（rotation値）が保持されたまま次のアニメーションが開始され、意図しない位置から回転が始まるため、1回目は正しく動作するが2回目以降で結果不一致が発生する

1.3 WHEN スロットマシンアニメーションを実行する場合 THEN 一番左のスロット（インデックス0）が完全に静止したまま回転せず、アニメーション開始時から結果が確定している

1.4 WHEN スロットマシンアニメーションで各列のアニメーションを開始する場合 THEN 最初の列（インデックス0）に対してcolumnDurationが負の値（duration - 0 * columnDelay = duration）となり、即座に停止する

### 期待される動作（正常）

2.1 WHEN 円形ルーレットアニメーションで1人以上を選出する場合 THEN 各回のアニメーションで針が指すユーザーが最終的な選出結果と完全に一致すること

2.2 WHEN 円形ルーレットアニメーションで1回目のアニメーションが完了した後、2回目以降のアニメーションを実行する場合 THEN rotation値がリセットされ、常に同じ初期位置から回転が開始され、すべての回で正しい結果が表示されること

2.3 WHEN スロットマシンアニメーションを実行する場合 THEN 一番左のスロット（インデックス0）を含むすべてのスロットが正常に回転し、順番に停止すること

2.4 WHEN スロットマシンアニメーションで各列のアニメーションを開始する場合 THEN すべての列に対して正の値のcolumnDurationが設定され、適切な時間をかけて回転すること

### 変更されない動作（リグレッション防止）

3.1 WHEN 円形ルーレットアニメーションの回転速度、イージング、視覚効果が適用される場合 THEN システムは引き続き既存の演出効果を維持すること

3.2 WHEN スロットマシンアニメーションで中央と右端のスロット（インデックス1, 2）が回転する場合 THEN システムは引き続き正常に回転と停止を行うこと

3.3 WHEN スロットマシンアニメーションで列ごとの停止時間差（columnDelay）が適用される場合 THEN システムは引き続き順次停止する演出効果を維持すること

3.4 WHEN 他のアニメーション方式（カードフリップ、ドラムロール）が実行される場合 THEN システムは引き続き正常に動作すること

3.5 WHEN ユーザーがアニメーション方式や時間を設定する場合 THEN システムは引き続き設定を正しく適用すること
